# MASTER IMPLEMENTATION PROMPT

**Landing / Marketing Site — Rigor**

---

## Context

You are a software architect responsible for designing the technical architecture of a marketing site for a conceptual AI framework named RIGOR.

RIGOR is not a SaaS startup brand.
It is a structural protocol initiative centered around constraint-based AI generation.

The site must reflect institutional authority, structural precision, and controlled communication.

Do not interpret this as a growth-oriented product website.
Interpret it as a formal declaration platform.

---

## Identity Constraints

The brand is defined by:

* Structural boundedness
* Deterministic framing
* Formal communication
* Institutional tone
* Dark-dominant visual system
* No decorative design patterns

The site must not resemble:

* Startup landing pages
* Growth SaaS templates
* AI hype branding
* Vibrant gradient-heavy interfaces

---

## Communication Constraints

The voice is:

* Declarative
* Precise
* Controlled
* Non-promotional

Avoid:

* Marketing exaggeration
* Emotional persuasion
* Conversational tone

All sections must follow:

Definition → Constraint → Mechanism → Outcome → Implication

---

## Visual System Constraints

* Dark background dominant
* Single accent color (deep blue)
* Strict grid alignment
* No gradients
* No rounded soft UI
* No drop shadows
* No illustration-heavy sections
* Diagrams preferred over decorative graphics
* Generous spacing

Typography:

* Institutional heading system
* Neutral body text
* Clear hierarchy

---

## Functional Scope

The landing site must:

* Present conceptual identity
* Explain AI Constraint Protocol (public overview)
* Establish differentiation
* Provide structural credibility
* Offer access to documentation (future-ready)
* Be scalable to include whitepapers or specifications

No dashboard.
No interactive demo required at v1.

---

## Required Sections

1. Hero (Declarative positioning)
2. Problem Definition
3. Structural Failure of Prompt Engineering
4. Introduction to AI Constraint Protocol
5. Conceptual Model (Diagram-ready)
6. Differentiation
7. Implementation Philosophy
8. Future Direction
9. Footer (Minimal, institutional)

---

## Technical Expectations

Architectural proposal must include:

* Recommended stack
* Scalability considerations
* Performance optimization strategy
* CMS vs static decision rationale
* Future extensibility plan
* Documentation hosting strategy
* SEO structure without marketing tone

---

## Output Format Required

Respond with:

1. High-level architecture proposal
2. Component breakdown
3. Folder structure
4. Deployment strategy
5. Justification aligned with brand constraints

Avoid generic SaaS assumptions.

Design must embody constraint.
