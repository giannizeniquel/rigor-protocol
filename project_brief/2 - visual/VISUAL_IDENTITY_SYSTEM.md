# VISUAL_IDENTITY_SYSTEM.md — v0.2

**Rigor — Visual Identity System Specification**

---

## 1. Purpose

This document defines the visual system of Rigor.

It establishes:

* Visual principles.
* Color system.
* Typography system.
* Layout structure.
* Spatial logic.
* Visual constraints.
* Accessibility compliance rules.

The visual identity must reflect structural boundedness, determinism, and restraint.

Design is not decorative.
It is structural expression.

Accessibility compliance is not optional.
It is structural validity.

---

## 2. Visual Principles

The visual system must embody:

* Containment
* Clarity
* Structural precision
* Deliberate restraint
* Controlled hierarchy
* Normative compliance

The interface must feel constrained, not expressive.

---

## 3. Color System

### 3.1 Base Palette

Primary background:

* Deep Black — `#0A0A0A`

Secondary surface:

* Structural Gray — `#1C1C1C`

Tertiary surface:

* Technical Gray — `#2A2A2A`

Primary text:

* Controlled White — `#F2F2F2`

Muted text:

* Muted Gray — `#A3A3A3`

---

### 3.2 Accent Color

Accent must be singular.

Recommended:

* Deep Blue — `#1E3A8A`

Accent usage rules:

* Used for structural emphasis only.
* Used in diagram boundaries.
* Used in underlines, borders, and structural markers.
* Never used as background fill for large surfaces.
* Not guaranteed for text usage if contrast requirements are not satisfied.

No gradient usage.
No multi-accent system.

---

### 3.3 Accessibility Compliance Rule

All text must comply with **WCAG 2 AA minimum contrast ratios**.

Minimum requirements:

* Normal text: **4.5:1**
* Large text (18px+ or 14px bold): **3:1**

Contrast compliance overrides accent preference.

If accent color fails contrast against a given background, it must not be used for readable text in that context.

Accessibility is a structural constraint.
It is not aesthetic preference.

---

## 4. Typography System

Typography must communicate institutional authority.

### 4.1 Heading Typeface

Requirements:

* Serif or semi-serif.
* High legibility.
* Moderate contrast.
* No ornamental styling.

Headings must feel foundational, not modern-startup.

Tracking slightly expanded for wordmark contexts.

---

### 4.2 Body Typeface

Requirements:

* Neutral sans-serif.
* Geometric but restrained.
* No rounded softness.

Body text must not compete with headings.

---

### 4.3 Typographic Hierarchy

Hierarchy must be strict:

* H1 — Large, authoritative, spaced generously.
* H2 — Section divider.
* H3 — Structural label.
* Body — Neutral, consistent.
* Caption — Muted gray, small scale.

No excessive font weights.

Avoid bold emphasis inside paragraphs unless structural.

---

## 5. Grid System

The grid must express containment.

* 12-column layout for desktop.
* Generous margins.
* Large vertical spacing between sections.
* Content blocks aligned strictly to grid.

No overlapping elements.
No floating decorative shapes.

All content must appear contained within invisible structural lines.

---

## 6. Spatial Logic

Spacing is structural communication.

Rules:

* Minimum section spacing: 96px vertical.
* Clear separation between conceptual blocks.
* Diagrams centered and isolated.
* No dense clustering of UI elements.

White space communicates boundedness.

---

## 7. Diagram Language

Diagrams are primary visual assets.

Characteristics:

* Monochromatic.
* Thin line weight.
* Rectangular boundaries.
* Explicit arrows.
* No curved or organic shapes.

Diagrams must resemble specification schematics, not marketing illustrations.

---

## 8. Iconography

Iconography must be minimal.

* Line-based.
* Geometric.
* Uniform stroke width.
* No filled icons.
* No emoji usage.

Icons are structural indicators, not decorative embellishments.

---

## 9. Motion System (If Applied)

Motion must express:

* Deterministic transition.
* State change clarity.
* Explicit directionality.

Allowed motion:

* Fade between states.
* Linear movement.
* Sequential reveal.

Forbidden motion:

* Bounce.
* Elastic easing.
* Decorative transitions.
* Parallax.

Motion must reinforce structural determinism.

---

## 10. CTA Design Philosophy

Calls to action must be restrained and accessible.

Primary CTA style:

* Text-based.
* No filled buttons by default.
* No decorative emphasis.

### 10.1 CTA Color Rule (Revised)

Primary CTA text color:

* Controlled White — `#F2F2F2`

Accent color must not be used as primary readable CTA text on dark background if it fails WCAG contrast requirements.

Deep Blue may be used for:

* Underlines.
* Left borders.
* Structural indicators.
* Hover state transitions.
* Icon accents.

### 10.2 CTA Accessibility Invariant

CTA text must always meet:

* Minimum 4.5:1 contrast ratio for normal text.

If contrast falls below this threshold, the implementation is invalid.

CTA is directional, not persuasive.
It must remain legible under all structural conditions.

---

## 11. Forbidden Visual Patterns

The following are prohibited:

* Gradients.
* Vibrant multi-color palettes.
* Rounded cards with heavy shadows.
* Soft-glass effects.
* Illustration-heavy sections.
* Abstract shapes.
* Startup-style hero imagery.

Rigor must not resemble a SaaS dashboard brand.

---

## 12. Visual Emotional Temperature

The visual system must feel:

* Calm.
* Controlled.
* Structured.
* Precise.

It must not feel:

* Playful.
* Energetic.
* Aggressive.
* Promotional.

---

## 13. Consistency Clause

The visual identity must remain consistent across:

* Marketing site.
* Documentation.
* Specification documents.
* Diagrams.
* Presentation materials.

Accessibility compliance is mandatory across all contexts.

Deviation weakens structural authority.

---

## 14. Evolution Constraints

The following are invariant:

* Dark-dominant base palette.
* Single accent system.
* Structured grid discipline.
* Diagram-centric communication.
* WCAG AA minimum contrast compliance.

Future visual evolution must preserve structural restraint and accessibility integrity.
